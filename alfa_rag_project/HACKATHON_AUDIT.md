# 🏆 Хакатон-Аудит RAG Пайплайна

## 🧠 Анализ узких мест (Chain of Thought)

### 1. Retrieval: Насколько эффективен чанкинг?

**РЕШЕНО:** `CHUNK_SIZE=450` (было 200)
- Теперь чанк = 3-5 предложений вместо 1-2
- `CHUNK_OVERLAP=100` (было 30) — достаточно для сохранения контекста на стыке

**Остаётся проблемой:**
- `TOP_K_RERANK=3` может пропустить релевантный чанк при плохом ранжировании
- Нет семантического перекрытия (только символьное)

### 2. Context Window: Не забьем ли мы контекст LLM лишним шумом?

**РЕШЕНО:**
- `MAX_SENTENCES=3` — основной лимит (по предложениям)
- `MAX_RESPONSE_CHARS=450` — safety лимит (не 150!)
- Это позволяет давать полные ответы на сложные вопросы (до 3x эталона)

### 3. Generation: Защищен ли системный промпт?

**РЕШЕНО:** Новый SYSTEM_PROMPT:
```
"Ты суровый банковский AI-аналитик. Отвечай на вопрос строго на основе предоставленного текста.

ПРАВИЛА:
1. Выдавай ТОЛЬКО факты из контекста. Никаких приветствий и лишних слов.
2. Если в контексте нет прямого ответа на вопрос, верни ровно два слова: "Недостаточно информации".
3. Отвечай максимально емко. Объединяй длинные списки в одно-два предложения через запятую.
4. Твой ответ не должен превышать 3 предложений."
```

### 4. Demo-safety: Что произойдет при ошибках?

**РЕШЕНО:**
- ✅ `timeout=30` добавлен во всех клиентах (OpenAI, OpenRouter)
- ✅ `try-catch` добавлен в `test_submission.py`
- ✅ Fallback на `extract_answer_from_context()` работает
- ✅ При ошибке LLM возвращается "Недостаточно информации"
