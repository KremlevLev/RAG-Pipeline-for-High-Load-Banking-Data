# Changes Made — Все сессии

## Сессия 1 — Критические фиксы

## 1. [`src/config.py`](src/config.py) — FIX-1: Адаптивная длина ответа

**Изменение:** Подняты лимиты длины ответа для соответствия реальному распределению эталонов.

| Параметр | Было | Стало | Почему |
|----------|------|-------|--------|
| `MAX_SENTENCES` | 2 | 5 | Эталоны содержат 3–6 предложений |
| `MAX_RESPONSE_WORDS` | 30 | 80 | Эталоны до 100+ слов со списками |
| `MAX_RESPONSE_CHARS` | 150 | 500 | 500 символов = баланс между 1.5x и 3x медианы эталона |

**Эффект:** устраняет главную утечку скора — жёсткая обрезка убивала BERTScore-Recall.

## 2. [`src/kaggle_main.py`](src/kaggle_main.py) — FIX-2, FIX-G1, FIX-G3, FIX-G6, FIX-3, FIX-4, FIX-5.1, FIX-6

### 2.1 SYSTEM_PROMPT (FIX-G1)
- Полностью переписан: убран императив «отвечай кратко/максимально ёмко/не превышай 3 предложений»
- Добавлено: «полно и фактически точно», «оформи списком, сохраняй ВСЕ пункты»
- Few-shot: заменён короткий пример на многошаговую инструкцию (получение карты)
- Добавлено: «НЕ пиши вводных слов (Согласно фрагменту, Таким образом)»

### 2.2 strip_preamble (FIX-G6)
- Добавлена функция для удаления мета-клише из ответов LLM
- Паттерны: «Согласно фрагменту N», «В фрагменте указано», «Таким образом», «Исходя из контекста»
- Удаление остаточных «(Фрагмент 3)» в конце ответа

### 2.3 KaggleGenerator.__init__ (FIX-2, BUG-5)
- Убрана передача `device_map` и `torch_dtype` в `pipeline()` — модель уже размещена через `AutoModelForCausalLM`, повторная передача вызывала риск двойного размещения / OOM на 2×T4

### 2.4 KaggleGenerator.generate (FIX-2, FIX-G3)
- `max_new_tokens=64` → **`max_new_tokens=320`** (было невозможно сгенерировать длинный ответ-список)
- Добавлено `return_full_text=False` в `pipeline()` — возвращает только сгенерированный текст
- **Удалён весь блок ручного вырезания промпта** (BUG-2): `split("</")[-1]`, проверка `prompt in answer` — больше не нужны
- Добавлен вызов `strip_preamble(answer)` перед truncate

### 2.5 AnswerCache (FIX-4)
- В ключ кеша добавлен `PIPELINE_VERSION` (сейчас `"v2-adaptive-len"`) — старые ответы не отравят новый инференс после изменений промпта/чанков
- `set()` больше не вызывает `_save()` на каждый вопрос — только устанавливает флаг `_dirty`
- Добавлен `save()` — вызывается раз в 2000 вопросов вместе с чекпоинтом

### 2.6 validate_answer (FIX-3)
- Валидация теперь **действует**: при невалидном ответе делает fallback на `extract_answer_from_context`
- Если fallback тоже не проходит — оставляет оригинал (логирует)

### 2.7 Резюм по q_id (FIX-6)
- Вместо опасного `idx < start_idx` — проверка по множеству `done_ids` (уже отвеченные q_id)

### 2.8 Колонка сабмита (FIX-5.1)
- Смена имени колонки с `answer` на `answer_new` (соответствует формату `sample_submission.csv`)

## Сессия 2 — Ретривер, Чанкер, Генератор

### 2.2. 5.3 — Порог «Нет ответа.» для нерелевантных вопросов

- [`config.py`](src/config.py): добавлен `MIN_RERANK_SCORE=0.05`
- [`retriever.py`](src/retriever.py): `get_context()` возвращает пустую строку, если лучший чанк ниже порога
- [`kaggle_main.py`](src/kaggle_main.py): `generate()` при пустом контексте отвечает «Нет ответа.» (совпадает с эталоном q_id=7,13)

**Эффект:** нерелевантные вопросы не гонятся через LLM — точное совпадение с эталоном повышает BERTScore на этих вопросах.

---

### 3. [`src/chunker.py`](src/chunker.py) — FIX-C1, FIX-C2

**Изменение:** `chunk_all_websites` переписан на класс `Chunker` с `clean_text()`.

| Параметр | Было | Стало |
|----------|------|-------|
| `ChunkerConfig.chunk_size` | 450 | 650 |
| `ChunkerConfig.chunk_overlap` | 100 | 120 |
| `ChunkerConfig.min_chunk_length` | 30 | 40 |
| `chunk_all_websites` имплементация | legacy `create_chunks()` (БЕЗ очистки) | класс `Chunker` с `clean_text()` |

**Эффект:** HTML, `&nbsp;`, `[chunk_id]`, служебные фразы вычищаются ДО индексации.
FAISS и BM25 строятся на чистом тексте, а не на мусоре.

### 4. [`src/generator.py`](src/generator.py) — FIX-G1, FIX-G8

- `SYSTEM_PROMPT` синхронизирован с `kaggle_main.py` (полнота + списки, без «кратко»)
- FIX-G8: добавлен `_precompute_doc_freq()` — расчёт IDF за один проход (O(N) вместо O(N²))
- `_compute_tfidf_score()` теперь принимает предрасчитанный `precomputed_idf: dict`, а не перебирает все предложения на каждом вызове
- В `extract_answer_from_context`: нормализованные предложения предрасчитываются один раз

### 5. [`src/retriever.py`](src/retriever.py) — FIX-R2/R3/R4/R5 + FIX-G4/G5

- **FIX-R5:** `CrossEncoder` прибит к конкретному GPU: `cuda:1` если ≥2 GPU, `cuda:0` если 1, `cpu` если нет
- **FIX-R3:** query проходит `normalize_for_embedding()` (ё→е, NFC) перед отправкой в FAISS — индекс и запрос теперь на одной нормализации
- **FIX-R4:** вместо «сырого» слияния — Reciprocal Rank Fusion (RRF) с K=60
- **FIX-R2:** reranker теперь скорит ОЧИЩЕННЫЙ текст (через `clean_chunk_text`), а не сырой HTML
- **FIX-G4:** простой реверс `[::-1]` заменён на зигзаг (лучшие чанки по краям, слабые в центре)
- **FIX-G5:** фрагменты нумеруются `[Фрагмент 1]`, `[Фрагмент 2]`... для cross-attention модели (удаляются `strip_preamble` в ответе)
